# Release Governance And Risk

## 1) Go/No-Go Framework

- Define mandatory checks for each release class.
- Define approval roles and decision deadlines.
- Keep release notes focused on behavior and risk changes.

## 2) Rollout Strategy

- Use staged rollout for high-impact changes.
- Define stop/rollback thresholds before release starts.
- Monitor leading indicators during rollout windows.

## 3) Rollback Readiness

- Keep known-good build references.
- Keep rollback commands and data migration implications documented.
- Verify rollback drills periodically.
