# Quality Model And Definition Of Done

## 1) Quality Dimensions

- Functional correctness.
- Security and privacy.
- Reliability and resilience.
- Usability and accessibility.
- Performance and efficiency.
- Maintainability and operability.

## 2) Definition Of Done

- Code and tests pass agreed quality gates.
- Observability and runbook impacts are updated.
- Risk and rollback criteria are documented.
- Ownership is assigned for post-release monitoring.
